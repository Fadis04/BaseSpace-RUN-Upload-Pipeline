BaseSpace RUN Upload Pipeline

Script Name: BS_run_upload_pip.sh
Author: Fadi Slimi
© 2025 Fadi Slimi – Medibio

Purpose: Upload NGS run folders to Illumina BaseSpace in a friendly and automated way, including interactive authentication, checksum verification, and logging.

1. Requirements

  1. Linux or macOS terminal with bash.
  2. Internet connection (required for BaseSpace CLI installation and authentication).
  3. BaseSpace account (login credentials required).
  4. NGS RUN folder ready for upload (includes Data, InterOp, RunInfo.xml, etc.).

2. Installation

You can run the script directly. It acts as a standalone tool.

  • It automatically checks if the BaseSpace CLI (bs) is installed.
  • If not found, it automatically downloads the latest version from Illumina, installs it to ~/bin/bs, and configures the path.

3. Running the Script

Open a terminal and run:

bash BS_run_upload_pip.sh

Step-by-Step Interactive Inputs

  1. RUN Folder Path:
    Enter the full path to the RUN folder:
    • Example: ~/runs/2025-11-17_NextSeq
    • The script validates that the folder exists.
  2. Run Name:
    Enter the name for this run: (Default: My_Run)
  3. Instrument Type:
    Enter instrument type: (e.g., MiSeq, NextSeq)
  4. Allow Invalid Read Names:
    Default is no. Type yes only if required.

4. Authentication (Smart Login)

The script handles authentication intelligently:

  • If you are already logged in: The script displays the current user (e.g., fadi_Medibio) and asks: 
    [AUTH] Currently logged in as: fadi_Medibio
Do you want to upload using this account? (yes/no):
    • Type yes to proceed immediately.
    • Type no to switch accounts (the script will launch the login process).
  • If you are NOT logged in: The script automatically launches bs auth and provides a link to log in via your browser.

5. Checksum & Upload

  • Checksums: An MD5 checksum file (md5sum.txt) is generated inside the run folder to ensure data integrity.
  • Upload: The script constructs the optimal upload command.
  • Confirmation: You will be asked to confirm before the upload starts: 
    Proceed with upload? (yes/no):

6. Logs & Errors

All activities are logged automatically to the ./logs/ directory.

  • Log format: logs/upload_YYYYMMDD_HHMM.log
  • Error Handling: The script checks for folder validity, installation success, and upload status. If an error occurs, check the log file for details.
