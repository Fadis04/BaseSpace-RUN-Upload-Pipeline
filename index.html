BaseSpace RUN Upload Pipeline

Script Name: BS_run_upload_pip.sh

Author: Fadi Slimi

© 2025 Fadi Slimi – Medibio

Purpose: Upload NGS run folders to Illumina BaseSpace in a friendly and automated way, including authentication, checksum verification, and logging.

1. Requirements

  1. Linux or macOS terminal with bash.
  1. Internet connection (required for BaseSpace CLI installation and authentication).
  1. BaseSpace account (login credentials required during first authentication).
  1. NGS RUN folder ready for upload (includes Data, InterOp, RunInfo.xml, etc.).
  1. Optional: wget installed (for CLI auto-install).

2. Installation

You can run the script directly. It will automatically:

  • Check if BaseSpace CLI (bs) is installed.
  • If not, download and install it from Illumina:

wget "https://launch.basespace.illumina.com/CLI/latest/amd64-linux/bs" -O ~/bin/bs
chmod u+x ~/bin/bs
export PATH="$HOME/bin:$PATH"

  • Verify the version:

bs --version

  • The script handles all this automatically for the user.

3. Running the Script

Open a terminal and run:

bash BS_run_upload_pip.sh

Step-by-Step Interactive Inputs

  1. RUN Folder Path:
    Enter the full path to the RUN folder:
  • Example: /home/user/test_run_2025-11-17
  • Supports ~ shortcut for home directory.
  • Script validates that the folder exists.
  • Script validates that the folder exists.
  1. Run Name:
    Enter the name for this run:

Default: My_Run if left empty.

  1. Instrument Type:
Enter instrument type (e.g., MiSeq, NextSeq):
  • Default: Unknown.
  • Examples: MiSeq, NextSeq2000, NovaSeq6000.
  1. Allow Invalid Read Names:

Allow invalid read names? (yes/no, default: no):
  • Default: no.
  • Use yes only if your run files contain non-standard read names.

4. Authentication

  • The script checks if you are already authenticated with BaseSpace CLI:
bs whoami
  • If not authenticated, the script launches:
bs auth
  • You will receive a URL to open in a browser for login. Once authenticated, BaseSpace CLI stores a token locally for future uploads.

5. Checksum Generation

  • The script automatically generates MD5 checksums for all files in the RUN folder:
find "$RUNFOLDER" -type f -exec md5sum {} \; > "$RUNFOLDER/md5sum.txt"
  • Ensures data integrity before upload.
  • Checksum file: md5sum.txt inside your RUN folder.

6. Upload Command

  • Constructed automatically:
bs upload run "<RUNFOLDER>" -n "<RUNNAME>" -t "<INSTRUMENT>"

Example:

bs upload run "/home/user/test_run_2025-11-17" -n "My_Run" -t "NextSeq2000"
  • Optionally adds -allow-invalid-readnames if user selected yes.
  • The script prompts before starting the upload:
Proceed with upload? (yes/no):

Type yes to start.

7. Logging

  • Logs are saved automatically in ./logs/ folder:
logs/upload_YYYYMMDD_HHMM.log

Contains:

  • Authentication details
  • Checksum generation
  • Upload progress
  • Error messages if any

8. Error Handling

  • If RUN folder is invalid → [ERROR] Folder not found!
  • If BaseSpace CLI fails to install → [ERROR] Failed to download BaseSpace CLI
  • If upload fails → [ERROR] Upload failed. Check the log file

9. Success Confirmation

After a successful upload, the script prints:

[INFO] Upload completed successfully!
  • Check logs to confirm all files were uploaded.

11. Notes

  • The script is fully automated for installation and authentication.
  • Users can distribute this script to clients who do not have BaseSpace CLI installed.
  • For advanced BaseSpace CLI commands: see official Illumina documentation:
    • bs upload run --help
    • bs auth --help
    • BaseSpace CLI Documentation https://developer.basespace.illumina.com/docs/content/documentation/cli/cli-introduction/
